# Инструкции по проекту binance-lab

## Что это
Личная лаборатория анализа крипторынка + бумажной торговли (paper trading) на данных Binance.
**Не** «бот, который зарабатывает». Цель по приоритету:
1. Непрерывный ежедневный ряд рыночных данных (топ-пары) в надёжном хранилище.
2. Честная симуляция стратегий на виртуальные деньги — измеряем P&L без риска.
3. Ежедневный разбор рынка (делает Claude).
4. Только если симуляция стабильно в плюсе — обсуждаем подключение реальных ~$10.

## Честные рамки (не забывать)
- Предсказательного эджа нет ни у кого. Любая закодированная стратегия — общеизвестна.
- $10 — не капитал, а «живой полигон». Комиссии 0.2%/круг + MIN_NOTIONAL ~$5-10 съедают мелкую прибыль.
- Реальные сделки — только после того, как paper trading честно отработал.

## Архитектура
- **Хостинг:** всё на Railway (не на домашнем NAS — там SPOF по электричеству). 24/7 без участия Томера.
- **БД:** Postgres на Railway. Таблицы: `candles`, `paper_trades`, `paper_equity`.
- **Collector:** Python-воркер, cron на Railway. Публичный API Binance (klines/ticker) — ключ НЕ нужен для рыночных данных.
- **Paper-движок:** применяет правила стратегий к новым барам, ведёт виртуальный портфель.
- **Деплой:** git push → Railway (как во всех проектах Томера). Никогда `railway up`.

## Данные (дефолты)
- Пары: BTCUSDT, ETHUSDT, SOLUSDT, BNBUSDT, XRPUSDT
- Таймфреймы: 1h (основной), 1d (контекст)
- История для старта: ~30 дней

## Безопасность
- Для рыночных данных API-ключ не нужен вообще — используем публичные эндпоинты.
- Если дойдём до реальной торговли: ключ ТОЛЬКО Spot Trading, withdrawals OFF, IP whitelist.
- Секреты — в Railway variables, не в репо.

## Стек
- Python 3.12, psycopg (Postgres), requests, pandas
- Railway (Postgres + cron worker)
